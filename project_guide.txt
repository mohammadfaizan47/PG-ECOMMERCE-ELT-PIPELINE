




STEP 1) → Create your GitHub Repository:

           • Repo name: REPO-NAME
		   • Description: REPO-RELATED
		   • Visibility: Public (showcasing to the recruiters)
		   • Initializing: Add README file
		   • .gitignore: select "Python" from dropdown
		   • License: MIT
		   
STEP 2) → Create a Structural theme on VS Code editor:

           • Open Powershell in your Desktop:
		         /~ cd PATH_WHERE_YOU_HAVE_CREATED_THE_FOLDER_LOCALLY
			     /~ code .
			   
		   • Open VS Code and Open Terminal of VS Code:
		      - Folders creation:
		         /~ mkdir dags
			     /~ mkdir tasks
			     /~ mkdir notebooks
			     /~ mkdir expectations
			     /~ mkdir config
			     /~ mkdir docs
			     /~ mkdir tests
			  - root-level files creation:
			     /~ New-Item .env.example -ItemType File
				 /~ New-Item Makefile -ItemType File
				 /~ New-Item requirements.txt -ItemType File
				 /~ New-Item project_guide.txt -ItemType File
				 
STEP 3) → Commit this structure to GitHub
           • Initializing Git and connecting to your GitHub repo:
                 /~ git init
                 /~ git remote add origin URL_OF_YOUR_GITHUB_REPO
                 /~ git fetch origin
                 /~ git checkout main
           • Now, push it
                 /~ git add .
                 /~ git commit -m "feat: initialize project structure with all folders and base files"
                 /~ git push origin main

           • If we want to modify any file and commit it via VS Code then we have to do it like:
                 /~ git checkout -b feat/your-file-name
                 /~ git add .
                 /~ git commit -m "feat: describe what you did"
                 /~ git push origin feat/your-file-name		

STEP 4) → Update .gitignore
           • Open .gitignore and add these lines at the bottom:
                 # Environment variables
                 .env

                 # Project personal notes
                 project_guide.txt

                 # Power BI cache
                 *.pbix

                 # Python cache
                 __pycache__/
                 *.pyc
                 *.pyo

                 # VS Code
                 .vscode/

                 # Airflow logs
                 logs/
                 airflow.db
                 airflow.cfg

                 # OS files
                 .DS_Store
                 Thumbs.db
           • Save it, then run it on terminal:
                 /~ git checkout -b feat/BRANCH_NAME
                 /~ git add .gitignore
                 /~ git commit -m "chore: update gitignore with project-specific exclusions"
                 /~ git push origin feat/BRANCH_NAME

           • To switch between branches:
                 /~ git branch
                 /~ git checkout main
                 /~ git checkout feat/BRANCH_NAME				 
			     /~ git branch -d feat/BRANCH_NAME (delete locally)
				 /~ git push origin --delete feat/BRANCH_NAME (delete on GitHub)
				 
STEP 5) → Fill in requirements.txt
           • Open requirements.txt in VS Code and add:
		         # Data generation
                 faker==37.1.0
                 psycopg2-binary==2.9.10

                 # AWS
                 boto3==1.38.0

                 # Data quality
                 great-expectations==1.4.4

                 # Testing
                 pytest==8.3.5
                 pytest-mock==3.14.0

                 # Utilities
                 python-dotenv==1.1.0
                 pyyaml==6.0.2
                 requests==2.32.3
				 
		   • Save it, then commit:
		         /~ git add requirements.txt
				 /~ git commit -m "chore: add project dependencies to requirements.txt"
				 /~ git push origin feat/project-setup
				 
STEP 6) → Create Config files

           • Open VS Code terminal and create these files:
		         /~ cd config
				 /~ New-Item dev.yaml -ItemType File
				 /~ New-Item prod.yaml -ItemType File
				 /~ cd ..
